● Das ist ein Bug. `lastFile` bleibt leer, obwohl eine Datei gedroppt
  wurde.

  **Ursache** liegt in `extractFilePathFromDrop` im JavaScript:

  ```javascript
  // Windows URI: file:///C:/Users/user/test.md
  var path = decodeURIComponent(line.replace(/^file:\/\//, ''));
  // Ergebnis: /C:/Users/user/test.md  ← führender Slash vor C:
  ```

  Auf Windows entfernt das Regex nur `file://` (2 Slashes), lässt aber
  `/C:/...` zurück. Daraufhin schlägt entweder:
  - `os.Stat("/C:/Users/user/test.md")` in Go fehl (Datei nicht
  gefunden), oder
  - `text/uri-list` wird von WebView2 gar nicht befüllt

  **Ich warte kurz bis der laufende Build fertig ist**, da der Agent
  gerade mehrere Dateien umbaut – dann integriere ich den Fix direkt
  mit.

  Der Fix:
  ```javascript
  var path = decodeURIComponent(line.replace(/^file:\/\//, ''));
  // Windows: /C:/Pfad → C:/Pfad
  path = path.replace(/^\/([A-Za-z]:)/, '$1');
  return path;
  ```
